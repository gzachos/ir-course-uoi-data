In computing, sleep is a command in Unix, Unix-like and other operating systems that suspends program execution for a specified time.

Overview[edit]

The sleep instruction suspends the calling process for at least the specified number of seconds (the default), minutes, hours or days.

The version of sleep bundled in GNU coreutils was written by Jim Meyering and Paul Eggert.[1] The command is also available in the OS-9 shell,[2] in the KolibriOS Shell,[3] and part of the FreeDOS Package group Utilities.[4] The FreeDOS version was developed by Trane Francks and is licensed under the GPL.[5]

A sleep command is also part of ASCII's MSX-DOS2 Tools for MSX-DOS version 2.[6]

In Windows PowerShell, sleep is a predefined command alias for the Start-Sleep cmdlet which serves the same purpose.[7] Microsoft also provides a sleep resource kit tool for Windows which can be used in batch files or the command prompt to pause the execution and wait for some time.[8] Another native version is the timeout command which is part of current versions of Windows.[9]

Usage[edit]

 sleep number

Where number is an integer[10] number to indicate the time period in seconds. Some implementations support floating point numbers.

Options[edit]

None.

Examples[edit]

 sleep 30

Causes the current terminal session to wait 30 seconds. 

 sleep 18000

Causes the current terminal session to wait 5 hours

GNU sleep[edit]

 sleep 3h ; mplayer foo.mp3

Wait 3 hours then play foo.mp3

Note that sleep 5h30m and sleep 5h 30m are illegal since sleep takes only one value and unit as argument. However, sleep 5.5h (a floating point[11]) is allowed. Consecutive executions of sleep can also be used. 

 sleep 5h; sleep 30m

Sleep 5 hours, then sleep another 30 minutes .

The GNU Project's implementation of sleep (part of coreutils) allows the user to pass an arbitrary floating point[11] or multiple arguments, therefore sleep 5h 30m (a space separating hours and minutes is needed) will work on any system which uses GNU sleep, including Linux.

Possible uses for sleep include scheduling tasks and delaying execution to allow a process to start, or waiting until a shared network connection most likely has few users to wget a large file.
